## Post Body

We code review infrastructure.

We code review configuration.

We code review monitoring rules.

We code review documentation.

We code review database migrations.

We even code review our CI pipelines.

But the architecture diagram?

That's a PNG someone exported from Lucidchart 6 months ago, sitting in a Confluence page nobody opens.

Diagrams are the last manual artifact in DevOps.

And it shows.

Every time an engineer joins a team and asks "do we have an architecture diagram?" — the answer is the same nervous laugh.

We can do better than this.

Question: when was the last time YOUR team's architecture diagram was actually accurate?
